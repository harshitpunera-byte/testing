===============================================================================
3. BACKEND STARTUP FLOW
===============================================================================

`python -m uvicorn app.main:app --reload`
  --> `app/main.py`
  --> configure CORS
  --> include routers
  --> startup `init_db()`
  --> validate DB connectivity
  --> if PostgreSQL:
      - verify Python pgvector package
      - `CREATE EXTENSION IF NOT EXISTS vector`
      - verify extension exists
  --> create tables
  --> ensure pgvector and GIN indexes


===============================================================================
4. UPLOAD FLOW
===============================================================================

Frontend
  [TenderUpload.jsx] / [ResumeUpload.jsx]
    --> build `FormData`
    --> POST upload route

Backend
  [document_ingestion.py]
    --> validate file
    --> compute SHA256
    --> lookup duplicate by `document_type + file_hash`
    --> if duplicate: reuse stored parse result
    --> else:
        - save file to `uploads/`
        - create `documents` row
        - extract page-aware text
        - clean pages
        - build semantic chunks
        - embed text
        - persist `document_chunks`
        - run structured extraction
        - normalize resume profile if resume
        - persist evidence rows
        - update document status to `stored`


===============================================================================
5. PDF EXTRACTION ORDER
===============================================================================

`load_pdf_pages(file_bytes, name)`
  --> PyMuPDF4LLM
  --> PyMuPDF
  --> pdfplumber
  --> Docling
  --> OCR via Tesseract

Meaningful-text checks
- strip whitespace noise
- strip image placeholder noise
- require real tokens


===============================================================================
6. CHUNK / EMBED / STORE FLOW
===============================================================================

cleaned pages
  --> semantic section detection
  --> chunking:
      - 800 words
      - 150 overlap
      - section + page metadata
  --> embeddings:
      - `BAAI/bge-small-en`
      - fallback hash embedding
  --> PostgreSQL storage:
      - `document_chunks.embedding`
      - `resume_search_index.summary_embedding`

Retrieval behavior
- semantic search via pgvector cosine distance
- lexical search via token overlap
- hybrid search via reciprocal rank fusion

===============================================================================
8. QUERY ROUTER FLOW
===============================================================================

`POST /match/`
  --> `query_service.answer_query()`
  --> build active document scope
  --> classify query intent
  --> mode:
      - `qa`
      - `matching`
      - `none`

QA
  --> hybrid retrieval over stored chunks
  --> load structured context
  --> prompt Ollama
  --> answer + sources

Matching
  --> load tender requirements
  --> derive better resume search query
  --> resolve candidate resumes
  --> normalize / repair candidate data
  --> deterministic score
  --> LangGraph reasoning summary


===============================================================================
9. STRUCTURED SEARCH FLOW
===============================================================================

`POST /search/resumes`
  --> parse constraints:
      - skills
      - title
      - location
      - minimum experience
      - maximum notice period
  --> classify mode:
      - semantic
      - structured_filter
      - structured_rank
      - hybrid
  --> query normalized resume tables
  --> optionally rerank with semantic search
  --> return paginated results


===============================================================================
10. SYSTEM HEALTH FLOW
===============================================================================

`GET /system/health`

  --> pgvector extension verification
  --> return:
