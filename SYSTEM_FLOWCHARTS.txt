Other user actions

1. Open PDF
User clicks "Open Resume" or "Open Tender" or profile row
  --> frontend builds `/documents/{id}/file`
  --> backend returns `FileResponse`
  --> browser opens stored PDF

2. Clear database
User clicks "Clear Database"
  --> confirm popup
  --> POST `/system/clear-database`
  --> backend deletes DB rows, uploads, vector files
  --> UI resets local state

Intent classifier keyword logic

  Query contains matching-style words?
  - match
  - shortlist
  - top candidate
  - best resume
  - find candidates
  - rank resume

If yes and both tender + resume exist
  --> mode = matching

Else
  --> mode = QA with scope:
      tender / resume / both

15. DETERMINISTIC SCORING FLOW
===============================================================================

  tender_data + resume_data
      |
      v
  [matching_service.py] -> `_score_candidate()`
      |
      +--> normalize phrases
      +--> match required skills against candidate skills
      +--> compute missing skills
      +--> match preferred skills
      +--> compare role
      +--> compare domain
      +--> compare candidate experience >= required experience
      |
      +--> numeric score:
      |      required skill score   = matched_required / total_required * 70
      |      preferred skill score  = matched_preferred / total_preferred * 10
      |      role score             = 10 if role match else 0
      |      domain score           = 10 if domain match else 0
      |      experience score       = 10 if candidate >= required else 0
      |      final score            = min(total, 100)
      |
      +--> verdict:
             if score >= 80 and experience_match -> Highly Suitable
             elif score >= 50                    -> Partially Suitable
             else                                -> Low Suitable


Scoring diagram

  Required Skills ---------------------- up to 70
  Preferred Skills --------------------- up to 10
  Role Match --------------------------- up to 10
  Domain Match ------------------------- up to 10
  Experience Match --------------------- up to 10
                                      ----------
                                      max 100

23. REAL CURRENT LIMITATION MAP
===============================================================================

These are not flow bugs in the diagram. They are the current design reality.

1. Gemini path is not active in code today.
  Docs mention Gemini, but `llm/provider.py` is currently Ollama-only.

2. Postgres is not active in default setup.
  SQLAlchemy supports non-SQLite DB URLs, but the shipped default is SQLite.

3. Docling is optional.
  `loader.py` imports it inside `try/except`, so the pipeline can run without it.

4. Embedding model is local-only.
  If `BAAI/bge-small-en` is not available locally, the app falls back to hashed
  embeddings.

5. LangGraph is minimal.
  The graph has one node, not a multi-agent orchestration pipeline.

6. Some files are placeholders or stale.
  They exist in the repo, but are not in the active runtime path:
  - `app/agents/resume_agent.py`
  - `app/agents/scoring_agent.py`
  - `app/agents/tender_agent.py`
  - `app/llm/extractor.py`
  - `tender-ui/src/App.css`
  - `tender-ui/README.md`
